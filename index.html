Data types and variables


Data types are a quite essential, yet simple concept to learn. There a few main datatypes:

  • Integers. Integers are typically reffered to as their shorter name, "int". An integer is what you think it is, an integer. Basically, it is either negative or positive, and it is always going to be a whole number. Attempting to add a decimal will throw and exception (error). Integers are more efficient than floats from a memory access speed and memory usage standpoint. Thus, it is good practice to avoid using a float unnecessarily.
  • Floats. A float is just like an integer, just it doesn't have the same rules. A float can store a decimal unlike integers. Not much more can be said about floats.
  • Double floating point. A double is a double precision version of the floating point variable. Although double percision means more digits in the decimal, that also means less efficient use of memory, which in a pinch or in larger usages, can be a bad thing for things such as games, 3D software, or RAM intensive applications. A double is not usually useful as floats already offer an entire 4 byte limit.

Another thing to take into consideration is a datatype modifier. A datatype modifier is something that modifies the length of a piece of data. The main datatype modifiers you will encounter are longs, shorts, signed, and unsigned. These datatype modifiers can be applied to any datatype. A long can be applied to a piece of data to lengthen the limit of that piece of data. Oh yeah, I forgot to mention, data has a limit. Even for temporary data. In some cases, when a variable is overloaded with digits, it will either throw an exception, or continue to work, yet silently, it will round everything to the nearest whole number for example, in the case of the floating point and double floating point datatypes. A short does the opposite of a long, and an unsigned variable may not caontain a negative, whereas a signed variable may. A last few words in this giant wall of text that this site is, will be about what a variable is. A variable is simply some data stored on the machine of the end user temporarily, that will change on demand when requested to do so, and can be accessed, read, or written to by software very quickly, at the cost of a lack of storage space and permanance. Each variable will be removed and cleared after each session. This means, if the user closes the application or reboots their machine, the data will be gone no matter what.



If, else, and loops.


Conditional statements and loops are like the bread and butter of C++. They are vital for a functional program in C++. A conditional statement can execute block of code under any condition. For example, what if you wanted a program to only play a sound when a timer has reached zero? You could use conditional statements. You could tell the computer, if the variable called timer has reached 0 or below that, play a sound. Take a look at the following code. That is how a conditional statement is formatted. The if tells the computer it is a conditional statement. Next, what's in the brackets is the condition. That is what the computer is waiting for so it can execute the code. Next, the curly braces denote where to code that will be executed is. Conditional statments are pretty simple. Now to look at some loops. One of the simpler loops is called the while loop. It is just like an if statement, but with a loop built-in. It is formatted the same way as a conditional statement, however, instead of if, you put while, and the code executed by a while loop will continue executing repeatedly until the condition in the parentheses is no longer true.
